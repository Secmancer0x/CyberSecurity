# Case Study: DSRM Password Change - Event ID 4794

## 🗂 Log Source
- **Provider:** Microsoft-Windows-Security-Auditing
- **Event ID:** 4794
- **System:** 2016dc.hqcorp.local (Domain Controller)

## 🧠 Description
This event indicates an attempt to change the Directory Services Restore Mode (DSRM) administrator password.

## 🔍 IOC & Context
- **User:** HQCORP\\administrator
- **SID:** S-1-5-21-...-500
- **Workstation:** 2016DC
- **Status:** Success (0x0)
- **Time:** 2017-06-09 19:21 UTC

## 🎯 MITRE ATT&CK
- **Tactic:** Persistence
- **Technique:** T1098.003 – Account Manipulation: Local Account

## 🚨 Detection Strategy
Trigger alerts for:
- Password changes on DSRM account
- Executed outside business hours
- Non-standard workstations or process IDs
- Repeated changes within short timeframes

## 🛡️ SIEM Query (KQL)
```kusto
SecurityEvent
| where EventID == 4794
| where Account != "authorized_admin"
| where TimeGenerated not between(datetime(08:00) .. datetime(18:00))
